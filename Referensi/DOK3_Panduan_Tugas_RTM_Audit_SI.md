## 📝 TUGAS 4 — PROJECT AKHIR AUDIT SISTEM INFORMASI (Kelompok)

### Informasi Dasar
- **Judul:** Tugas ke-4: Project Akhir Studi Kasus Audit Sistem Informasi
- **Bentuk:** Kelompok (4–5 orang per kelas)
- **Waktu pengerjaan:** Mulai dipersiapkan dari Pertemuan 1–2, dikerjakan Pertemuan 11, dipresentasikan Pertemuan 11–15
- **CPMK:** CPMK124 — Mampu melaksanakan prosedur, memberikan rekomendasi dan menyusun hasil audit TI

### Apa yang Harus Dibuat?

**Laporan Project Audit Sistem Informasi** pada sebuah perusahaan/organisasi nyata menggunakan framework COBIT (4, 5, atau 2019), kemudian **dipresentasikan** di depan kelas.

---

### 📋 Template Laporan (WAJIB ikuti format ini)

```
BAB I — PENDAHULUAN
  A. Latar Belakang
  B. Permasalahan
  C. Tujuan

BAB II — PEMBAHASAN
  A. Framework yang digunakan (COBIT 4, 5, atau 2019)
  B. Domain dan Subdomain yang digunakan
  C. Capability Level / Maturity Level dan GAP

BAB III — PENUTUP
  A. Kesimpulan
  B. Saran
```

---

### 📋 Langkah-Langkah Pengerjaan Project

**FASE 1 — Persiapan (Sebelum Pertemuan 9)**
1. Bentuk kelompok 4–5 orang
2. Diskusikan perusahaan/organisasi yang akan diaudit (bisa fiktif atau nyata)
3. Pilih framework: COBIT 4.1, COBIT 5, atau COBIT 2019
4. Tentukan domain dan sub-domain yang relevan dengan masalah perusahaan

**FASE 2 — Penyusunan Laporan (Pertemuan 9–10)**
1. Tulis BAB I: Latar Belakang (kenapa perlu audit?), Permasalahan, Tujuan
2. Tulis BAB II awal: Jelaskan framework, domain, dan sub-domain yang dipilih
3. Buat kuesioner berdasarkan sub-domain yang dipilih
4. Sebar kuesioner ke responden

**FASE 3 — Pengolahan Data (Pertemuan 10–11)**
1. Tabulasi jawaban kuesioner
2. Hitung maturity/capability level sesuai framework yang dipilih:
   - **COBIT 4:** Hitung rata-rata nilai per sub-domain → rata-rata per domain → GAP
   - **COBIT 5:** Hitung Indeks Kuesioner → Maturity Index → Maturity Level → GAP
   - **COBIT 2019:** Hitung skor & mean → % ketercapaian → Rating (N/P/L/F) → Capability Level → GAP
3. Analisis GAP (selisih antara kondisi saat ini dan kondisi yang diharapkan)
4. Tulis BAB II lanjutan: Hasil perhitungan dan analisis
5. Buat rekomendasi mitigasi/perbaikan
6. Tulis BAB III: Kesimpulan dan Saran

**FASE 4 — Presentasi (Pertemuan 11–15)**
1. Buat slide presentasi yang sistematis
2. Presentasikan di depan kelas dengan penjelasan alur framework
3. Siapkan diri untuk menjawab pertanyaan
4. Alternatif: bisa juga dibuat dalam bentuk **video penjelasan**

---

### 🔢 Panduan Memilih Domain (Contoh)

**Jika memakai COBIT 4.1:**
- Pilih domain dari: PO (Planning & Organize), AI (Acquire & Implement), DS (Deliver & Support), ME (Monitor & Evaluate)
- Contoh: PT. XYZ menggunakan PO6, PO7, PO8, DS7, ME1, ME2

**Jika memakai COBIT 5:**
- Pilih domain dari: EDM, APO, BAI, DSS, MEA
- Contoh: Universitas XYZ menggunakan domain BAI (BAI01–BAI10)
- Contoh: UMKM MAM menggunakan EDM04, APO04, BAI08, DSS01, MEA03

**Jika memakai COBIT 2019:**
- Pilih domain dari: EDM (5), APO (14), BAI (11), DSS (6), MEA (4)
- Contoh: RS XYZ menggunakan EDM03, APO12, APO13, APO14, DSS05

**Tips memilih domain:** Sesuaikan dengan masalah yang dihadapi perusahaan:
- Masalah keamanan → pilih DSS05, APO13, EDM03
- Masalah SDM → pilih APO07, PO7
- Masalah inovasi → pilih APO04
- Masalah manajemen risiko → pilih APO12, EDM03
- Masalah data → pilih APO14

### 🔢 Rumus Perhitungan (Ringkasan)

**COBIT 4 — Maturity Level:**
```
Nilai sub-domain = Rata-rata skor jawaban responden per sub-domain
Nilai domain = Rata-rata semua sub-domain dalam satu domain
GAP = Expected Level – Current Level
```

**COBIT 5 — Maturity Level:**
```
Indeks Kuesioner = Σ Jawaban Kuesioner / Σ Pertanyaan
Maturity Index = (% WP Aktual / WP Standar) × Indeks Kuesioner
Maturity Level = Σ Maturity Index / Jumlah Sub-domain
Rating: N(0–15%), P(15–50%), L(50–85%), F(85–100%)
```

**COBIT 2019 — Capability Level:**
```
Skor = Σ (Jawaban Responden × Skala)
Mean = Total Skor / Jumlah Responden
% Ketercapaian = (Skor / Skor Maksimal) × 100%
Rata-rata per Level = Rata-rata % ketercapaian semua pertanyaan dalam level yang sama
Rating: N(0–15%), P(15–50%), L(50–85%), F(85–100%)
Capability Level = Level tertinggi yang mencapai L atau F
GAP = To-be Level – As-is Level
```
